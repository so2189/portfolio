    • Number Line ML

    An agent-based simulation and genetic algorithm optimization project where multiple agents traverse number lines probabilistically. The goal: evolve a strategy that minimizes the total movement variance before any agent reaches the goal.

    Visit Github

    Overview

    In Number Line ML, four independent agents start at position 1 on separate number lines ranging from 1 to 10. The simulation evolves a strategy to decide which agent to move at each step using a genetic algorithm.

    Game Rules

    • Each agent begins at position 1 on a 10-position number line.
    • At each time step, a single agent is chosen to move:
      • If at position 1 → always move to position 2.
      • Otherwise → move left or right with 50% probability each.
    • The game ends when any agent reaches position 10.
    • The score is the sum of squares of the number of moves each agent made. Lower scores are better.

    Optimization Approach

    A genetic algorithm is used to evolve agent-picking strategies. Policies are evaluated across simulations based on their final scores. The lower the score, the more balanced and efficient the strategy.

    Core Components

    • Environment: Simulates agents and number lines.
    • State Encoding: Combines current position and number of moves into a state.
    • Policy: Maps states to action values using a lookup table with random initialization.
    • Evaluation: Repeated simulation to calculate average performance.
    • Genetic Operators: Mutation and crossover drive exploration and refinement.

    Outcomes

    • Clear learning trajectory visible through decreased fitness scores over generations.
    • Strategic agent selection emerges naturally from policy evolution.
    • Successfully balances fairness (equal agent movement) with goal completion efficiency.

    Potential Extensions

    • Extend to more agents or longer number lines.
    • Apply reinforcement learning with Q-learning or policy gradients.
    • Introduce stochastic reward systems or penalties to increase complexity.
